# R1a design audit findings (18) — against built code

1. [must-fix] C1-5 Fireworks (web/src/lib/scenes/ch1/Fireworks.svelte + ch1/index.ts): the storyboard's set piece — IPL sixes lift out of the wall in two phases and re-sort into 19 per-season columns where 'each point is a real six', with step 3's two-tone as a per-point recolor (the attrs.u8 top-10-hitter bit, storyboard §8/§12 must-fix #4) — is not built. Shipped: a small in-place highlight lift (lift 0.14, othersDim 0.12) plus an abstract SVG bar chart carrying the counts and the two-tone. The caption still ships the storyboard's exact line 'Every IPL six ever hit, stacked by season' — but the pixels it describes are 19 rects, not stacked sixes. This is precisely the pixels-vs-copy class the storyboard's revision notes elevated to must-fix (§12 item 6), and the per-point recolor spec is silently dropped.
   FIX: Two sanctioned paths: (a) request shell support per CONTRACT §7 for a subset re-sort (six-class points to season columns, reversible), restoring the authored choreography and the per-point two-tone; or (b) get owner sign-off to keep the 2D chart and re-scope the copy so it no longer claims the bars are the sixes (e.g. 'Every IPL six ever hit, counted by season' or 'IPL sixes per season'), record the decision in storyboard §11, and delete the now-dead attrs.u8-bit pipeline requirement from §8. Either way, add the pixels-match-copy QA line to the release gate for this scene.

2. [must-fix] Mobile SVG label legibility (OutRate.svelte, WplBeat.svelte, Fireworks.svelte): all three ch1 charts use fixed 640-unit-wide viewBoxes with 10–11px SVG text. At width: min(680px, 94vw) on a 375px phone the render scale is ~0.55, so every axis label, the era end-of-line labels ('2008-10' / '2023-26' / 'WPL'), the tappable tick numbers, and the y-axis '%' labels render at ~6px effective — illegible. The era-ghost convention (the chapter's key comparison device) depends entirely on those end labels; on mobile the out-rate shock and the C1-6 peer-panel disambiguation both lose their labels. Mobile-first is a standing rule.
   FIX: Scale SVG type up in user units under a width breakpoint (e.g. text at 16–18 viewBox units below 640px via a CSS class on the svg), or move axis/end labels to HTML overlays positioned in CSS px (as Wall.svelte already does for its scaffold labels), or ship a narrower mobile viewBox. Verify at 320px and 375px widths; target ≥11px effective for all data labels.

3. [must-fix] CO-3 counter-stop staging (Assembly.svelte lines 92–115): buffer chronology puts the WPL's March-2023 points before IPL 2023, so the code fires the pinned 'And a second league assembles — deliberately apart…' caption FIRST, then '2023: the year the ceiling broke.' ~5k points (~1.6% of scrub, well under the storyboard's '~10vh beat later') afterwards — the reverse of the authored order (§12 should-fix: 2023 stop staged a beat BEFORE the WPL start). Consequences: the WPL card's leading 'And' continues a sentence that hasn't happened; the ceiling micro then renders SIMULTANEOUSLY above the still-pinned WPL card (micro-captions and the pinned card are separate slots in .caption-col), so two caption cards + two changes share one beat — violating the one-change rule the staging existed to protect. The code comment acknowledges the reversal but the shipped result is neither the storyboard nor a coherent re-authoring.
   FIX: Preferred: pipeline emits season-blocked point order with IPL 2023 before WPL 2023 (assembly is 'season by season'; within-season order is a pipeline choice), restoring the authored sequence with no scene changes. Alternative: re-author for the true order — rewrite the WPL caption to stand alone (drop 'And'), suppress the ephemeral micro while a pinned card is visible (one card at a time), space the ceiling stop a real beat after the WPL pin, and record the new order in storyboard §11 for owner sign-off. Also fix the ceiling copy's trailing punctuation ('broke —' vs shipped 'broke.') to whichever order wins.

4. [should-fix] Capped-bucket labels: OutRate.svelte (ticks array line 85) and WplBeat.svelte (line 99) label the right-edge x tick '30' while the wall labels the same bucket '30+'. Storyboard C1-4/C1-6 spec the shared x convention as 'balls 1–30+', and §10 QA has a release-blocking line: 'every capped bucket labeled 30+'. An open bucket is masquerading as a plain value on the two charts that share the wall's axis.
   FIX: Render the final tick label as '30+' in both charts (and in their aria-labels), after confirming with the pipeline that index 30 of hazard_pct_by_ball_index / sr_by_ball_index is the capped 30+ bucket. If it is genuinely 'exactly ball 30', document that divergence from the wall's convention in the storyboard instead.

5. [should-fix] Chapter nav visibility contradicts storyboard §6 (ChapterNav.svelte + +page.svelte line 44): §6 says the ☰ button is 'visible in every scene… dims to 40% opacity during the assembly scrub and the ignition morph; never hidden'. Shipped: for first-time readers the button does not render at all until the reader leaves the cold open (navVisible = chapter !== 'coldopen' || deepEntry || resume), and no set-piece dimming exists anywhere. CONTRACT §6 ('appears after the cold open') and the storyboard disagree; the storyboard is implementation truth.
   FIX: Either implement §6 — always render the button, add a .dimmed (opacity 0.4) state while the active scene is co-assembly or ch1-wall and its morph is in flight — or get the owner to amend storyboard §6 to the contract's 'appears after the cold open' behavior. Don't leave the two sources contradicting each other.

6. [should-fix] CO-2 is silently skippable into nothing (Draw.svelte): a reader who scrolls past the draw screen without tapping Reveal OR Just-show-me never sees the truth line, the gap shading, or the universal bridge line ('Nineteen seasons. 1,331 matches. Here comes every ball of it.' — specced for ALL paths). The cold open's factual payload (the 200s explosion) and the authored hand-off into the assembly are gated behind a tap; CONTRACT §8.3 says interaction adds depth, never carries a beat. The storyboard sanctions the tap gate for the *reveal animation*, not for losing the beat entirely.
   FIX: Auto-resolve on scroll-past: when the draw scene's progress crosses a late threshold (e.g. 0.85) with phase still 'draw', run the just-show-me path (write {skipped:true}, render the truth instantly, show the bridge line) so every reader exits the cold open having seen the truth and the bridge. Keep the buttons as the primary path.

7. [should-fix] Footnote layer has no route to the methods page (footnotes.ts + FootnotePanel.svelte): storyboard CO-3's ball-count footnote ends with 'Link: Full methods →', and §5 says the methods page is 'linked from EC + footnote sheets'. FootnotePanel renders plain-text paragraphs only, and no registry entry carries a link — the progressive-disclosure chain (main flow → sheet → methods) is severed at its last hop everywhere except the end card and nav.
   FIX: Add a standing footer inside FootnotePanel — a 'Full methods →' link to `${base}/methods/` rendered on every sheet (keeps the registry plain-text and satisfies the spec for all entries at once). It also becomes a focusable element, fixing the panel's currently link-free tab cycle.

8. [should-fix] WPL payoff card parses prose with a regex (picker/PayoffCard.svelte line ~102): the small-sample honesty sentence is extracted from the JSON headline via /points of intent\.\s+(.+)$/. If the pipeline rewords the headline (or a negative-delta variant like DC-W formats differently), extraction fails and the card renders the hand-templated league clause PLUS the full JSON headline — duplicated/conflicting claims, and the honesty sentence loses its specced position between the league clause and the team pair (storyboard C1-7 snapshot-review note). Copy-as-API via regex is fragile for a snapshot-tested surface.
   FIX: Pipeline payoff/ch1.json v2: ship `honesty` (and ideally the team-pair sentence) as discrete fields per variant; template renders fields only. Add a snapshot-harness assertion that the honesty field is non-empty for every small_sample variant, replacing the regex fallback.

10. [should-fix] Fixed footnote affordance vs mobile caption slots: the shell's 'ⓘ how we computed this' button is fixed at bottom 14px (~44px tall).    FIX: Apply Assembly's clearance formula to the ch1 mobile caption slots (bottom: max(72px, env(safe-area-inset-bottom) + 60px)), or reserve the bottom-right corner: cap caption width so the card never extends under the affordance.
